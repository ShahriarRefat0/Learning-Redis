# 🐳 Docker Interview Questions

## 1. What is Docker?

Docker is an open-source platform that enables developers to build, package, and run applications in lightweight containers. Containers ensure consistency across development, testing, and production environments.

---

## 2. What is the difference between Docker and Virtual Machines?

| Docker                | Virtual Machine          |
| --------------------- | ------------------------ |
| Shares host OS kernel | Requires a full guest OS |
| Lightweight           | Heavyweight              |
| Faster startup        | Slower startup           |
| Uses less resources   | Uses more resources      |

---

## 3. What is the difference between a Docker Image and a Docker Container?

### Docker Image

A read-only template containing application code, dependencies, libraries, and configurations.

### Docker Container

A running instance of a Docker image.

Example:

```bash
docker run nginx
```

This command creates and starts a container from the `nginx` image.

---

## 4. What is a Dockerfile?

A Dockerfile is a text file containing instructions used to build a Docker image.

Example:

```dockerfile
FROM node:22-alpine

WORKDIR /app

COPY . .

RUN npm install

CMD ["npm", "start"]
```

Build the image:

```bash
docker build -t my-app .
```

---

## 5. What is Docker Compose?

Docker Compose is a tool used to define and manage multi-container Docker applications using a `docker-compose.yml` file.

Example:

```yaml
services:
  app:
    build: .

  redis:
    image: redis:7-alpine

  postgres:
    image: postgres:17
```

Run all services:

```bash
docker compose up -d
```

---

## 6. What are Docker Volumes?

Docker Volumes are used to persist data generated by containers. Data remains available even if a container is removed.

Example:

```bash
docker volume create postgres_data
```

---

## 7. What is the difference between `docker ps` and `docker ps -a`?

* `docker ps` → Shows only running containers.
* `docker ps -a` → Shows all containers, including stopped ones.

---

## 8. What is Port Mapping in Docker?

Port mapping exposes a container port to the host machine.

Example:

```bash
docker run -p 3000:3000 node-app
```

Host Port → `3000`

Container Port → `3000`

---

## 9. How can you view Docker logs?

```bash
docker logs <container_name>
```

Example:

```bash
docker logs redis
```

---

## 10. What command is used to stop and remove a container?

Stop container:

```bash
docker stop <container_name>
```

Remove container:

```bash
docker rm <container_name>
```
